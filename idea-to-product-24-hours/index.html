From Idea to Product in 24 hours!

I’ve been working as a freelancer for a while now. It’s exciting to quickly prototype and deliver a project. Usually, my applications are used in exhibitions for a short time or purely for internal use, but some of my favourite projects are the ones that have been published online for everyone to see and enjoy. unfortunately, such projects are rare. That’s why I’ve been thinking of working towards developing more projects to share online.

To create software one needs to solve a problem. It’s easy to find a problem to work on, but usually, the easiest problems to solve have many solutions already implemented and available. Recently I experienced a very weird problem. I bought a keyboard but didn’t like how little noise the keys were making when typing. I wanted some instant sound feedback and decided it would be fun to install some software that would play sounds when I press certain buttons.

After a quick search, I realised this problem has been solved by numerous software solutions. This meant that there’s a need for such software. When you think about it, installing software to play sounds when you press on keyboard buttons is a bit of a security risk. The application will be able to log all your keypresses and potentially leak the information to the developer that made it. I will never install a software solution that does keypress tracking and is not open-sourced! It’s hard enough already to prevent malicious keyloggers installing in the background, why would I install one myself?

I couldn’t find any open-sourced applications that play a sound when you press keyboard buttons, so I made one: Link

I’d like to stress out that this is neither the cleanest implementation possible, nor it’s a difficult problem to solve. The main goal of this project was to practice not the development process, but the software publishing aspect of it.

App development

To create “Open Typewriter” I used Python. I decided to use Python because I’ve been joyfully using it recently. To create .exe out of the .py file I used Pyinstaller.

It’s a very neat tool and I made a simple .bat file that generates an executable:

pyinstaller --onefile --noconsole --clean --distpath="_exe" --icon=assets/icons/onIcon.ico OpenTypewriter.py

xcopy /s assets _exe\assets\

Even tiny projects like this one come with unforeseen challenges. The most tricky challenge with this project was the sound playback. I had some trouble finding the right solution that plays instantly and supports multiple files at the same time. After trying multiple available libraries I decided to use playsound. It worked well, but it had a memory leak issue. I had to modify the source code and set up a custom buffer to deal properly with the sound files after they’ve been played.

For keyboard press events I used keyboard. This worked nicely and let me set up short-cuts for toggling OpenTypewriter.

The first iteration of the app worked with a console window opened. Of course, this is not acceptable, so I disabled it in pyinstaller. I wanted to have some form of control the user can use to stop or disable the application. I decided to use pystray which creates a neat system tray icon. You can either right-click it to toggle the application or left-click it to open up a menu.

The final issue I had to solve was that the application can be triggered multiple times. This was adding multiple app instances working in parallel, with each one having their tray icon as well as sound effects. To prevent this I used the tendo singleton.

Open-source I open-sourced “Open Typewriter” on Github. I wanted to set the project properly for versioning, so the first version was pushed in a branch called v1.0.0. I’ve also set-up a release for that version with the executable as a download: https://github.com/grigtod/OpenTypewriter/releases

I’ve set up Open Typewriter with MIT License, so feel free to tinker with it and add more features. Perhaps we should add more sound packages with different effects?

Website

I distribute the application via a website. I used a simple one-page template from here: https://html5up.net/

I edited the CSS style and index.html to have aesthetics similar to the app icon. Then I added links to the Github source code, the GitHub executable release and a Paypal donation page.

When I was working on the website I got an idea that it would be nice if I can demonstrate how the application works on the webpage. I used some javascript code, so users of the website will hear the same typewriter sounds they’d hear if they used the app.

For web hosting, I’m using GitHub Pages, which is essentially another GitHub project with the website files: https://github.com/grigtod/OpenTypewriterWebsite
