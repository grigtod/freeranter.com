How I left Squarespace to speed up my website

I’ve been very annoyed by the webpage bloat experienced nowadays, especially on mobile. If your page takes more than 5 seconds to load on an average smartphone you’re doing it wrong!

I didn’t want to be a part of the problem and decided to rework my portfolio website to be more lightweight.

I was using Squarespace in the past. Squarespace tends to be very bloated, super slow when loading on mobile and cost me £144 per annum. I spent 2 days creating new websites. I divided my projects in two - company and freelance websites.

  • All freelance projects can now be found in my portfolio website: grigtod.com.
  • All personal projects can now be found in my company website: notelos.co.

Both websites use the same design with a few small changes in the header, footer and the main page. The new design is not much different from the old design, it’s just more lightweight and doesn’t include any tracking and useless javascript. For hosting, I’m now using Github Pages, which is free. For my “secret” projects page I’m using a 256-bit encryption that’s decrypted on the user’s PC when the correct password is provided. This makes it possible for me to host it in the github repo with the main website, but prevent unauthorized users from accessing the information. Even the images in the “secret” page are converted to data URI, which is a base64 string that contains the image contents. The images are thus contained in the encrypted webpage.

To measure performance I decided to use the “Lighthouse performance scoring” from Google Page Insights. This score tends to fluctuate between tests, but I want to demonstrate a performance increase on average. My website now loads much faster and uses fewer megabytes.

The desktop average performance changed from 81 to 98 (+17 points). Not much different, but even small speed-ups count toward a better user experience.

The mobile performance changed from 26 to 80 (+54 points). This is a substantial performance improvement!
